# Ferrosonic Audit 2026-05-13

Audit produced by four parallel deep-read passes over `src/daemon`,
`src/audio`, `src/ipc`, `src/config`, and `src/app`. Findings ranked by
severity. Each item lists file + line range, trigger, and a fix sketch.

Tracking column: `[ ]` open, `[x]` fixed, with the commit short hash.

---

## HIGH severity

### H1. TOCTOU in `play_queue_position` [x]
- **File:** `src/daemon/core.rs` L672-708
- **Trigger:** concurrent queue mutation (Remove/Move/Shuffle/Replace)
  between the read-lock snapshot of `queue[pos]` and the later
  write-lock assignment of `now_playing.song`.
- **Effect:** `now_playing.song` desyncs from `queue[queue_position]`.
- **Fix:** take one write lock spanning both the read of `queue[pos]`
  and the state mutation, or re-validate by song id after re-locking.

### H2. TOCTOU in gapless-advance branch [x]
- **File:** `src/daemon/core.rs` L1207-1240
- **Trigger:** queue mutated by IPC between the read-lock snapshot of
  `(next_pos, song)` and the later write-lock + `playlist_remove(0)`.
- **Effect:** wrong-song display, possible mpv index drift if a future
  change appends two preloads.
- **Fix:** hold the write lock across the whole advance block; re-fetch
  mpv `playlist-pos` under the same mpv lock before `playlist_remove`.

### H3. `restore_queue_blocking` is not blocking [x]
- **File:** `src/daemon/core.rs` L131-143
- **Trigger:** an IPC `Play` request arrives between `DaemonCore::new`
  returning and the spawned restore task acquiring the state write
  lock.
- **Effect:** play sees empty queue, returns Ok(()) silently; restore
  then overwrites the queue with the snapshot, eating the user's
  input.
- **Fix:** load the snapshot synchronously inside `new_with_mpv` before
  building the `Arc`, or block the IPC server on a `Notify` the restore
  task signals.

### H4. Direct-mode idle window [x]
- **File:** `src/daemon/core.rs` L1242-1264
- **Trigger:** same class as the just-fixed Buffered race. `loadfile`
  returns before mpv flips `idle-active=false`. Periodic poll sees
  state=Playing, mpv idle, fires `advance_auto`.
- **Effect:** Direct-mode play (manual Next, manual click within queue)
  can skip to the next track.
- **Fix:** mark a "just-loaded" instant under the mpv mutex on every
  `loadfile`; idle-advance branch ignores idle within ~1s of that
  instant. Or use the same per-task gate pattern.

### H5. Early-skip spurious fire after Buffered stop [x]
- **File:** `src/daemon/core.rs` L1163-1186
- **Trigger:** stale `position` (last polled before stop) combined with
  the new song's `duration` yields `time_remaining` in (0, 2.0). Mpv
  playlist count is 0 or 1 after the Buffered stop. Manual skip fires.
- **Effect:** the just-requested song gets immediately advanced past.
- **Fix:** gate the early-skip on the new `prebuffer_loading` slot or
  require `position > 0.5` so a fresh load can never trigger it.

### H6. `SubsonicClient` clones outlive server reload [x]
- **File:** `src/daemon/core.rs` L62-85 and every callsite that does
  `subsonic.read().await.clone()`.
- **Trigger:** user reconfigures server while a library/play request
  is in flight.
- **Effect:** in-flight call uses old client, commits results into new
  server's `state.library`; stream URLs can point at the old server.
- **Fix:** add `config_generation: AtomicU64`. Handlers capture the gen
  at start, discard their result if the gen changed. Or serialize
  reloads behind a mutex.

### H7. `MpvController::send_command` has no overall deadline [x]
- **File:** `src/audio/mpv.rs` L196-218
- **Trigger:** mpv hangs (audio device init, disk stall). The 100ms
  read timeout is per-read and the loop does `continue` forever.
- **Effect:** mpv mutex held forever, whole audio path freezes.
- **Fix:** wrap the read loop in an overall budget (e.g. 5s Instant
  deadline). Better: split reader into a dedicated task with a
  request-id demux table so events can drive advance too.

### H8. Stale-response correlation in mpv reader [ ]
- **File:** `src/audio/mpv.rs` L175, L196-218
- **Trigger:** early-return paths leave previous response/event lines
  unread; next `send_command` discards them as non-matching.
- **Effect:** brittle; future risk more than current bug.
- **Fix:** dedicated reader task + `oneshot` map keyed by request_id.

### H9. PipeWire `current_rate` cache diverges from reality [x]
- **File:** `src/audio/pipewire.rs` L67-95
- **Trigger:** any external `pw-metadata` change between our last set.
- **Effect:** subsequent `set_rate` short-circuits on cache-equals,
  PipeWire stays at the wrong rate. Breaks bit-perfect playback.
- **Fix:** drop the cache short-circuit, or re-query `pw-metadata`
  before comparing.

### H10. Per-connection requests handled concurrently [x]
- **File:** `src/ipc/server.rs` L126-135
- **Trigger:** one client sends Request A then Request B quickly.
- **Effect:** A and B reach `DaemonCore` in undefined order via
  separate `tokio::spawn` tasks; response order on the wire can differ
  from request order.
- **Fix:** await dispatcher inline on the read loop, or serialize with
  a per-connection mpsc.

### H11. `EnqueueMode::Replace + play_from` is not atomic [x]
- **File:** `src/ipc/client.rs` L280-293
- **Trigger:** another client mutates the queue between Replace's lock
  drop (L285) and `play_queue_position(Buffered)` re-acquiring.
- **Effect:** the song at index `play_from` may not be from the songs
  A enqueued.
- **Fix:** lock once, swap queue, capture target `Child` by clone, call
  a daemon helper that plays the captured `Child` directly.

### H12. Broadcast Lagged drops events with no resync [x]
- **File:** `src/ipc/server.rs` L97-106
- **Trigger:** TUI stalls past the 256-event ring.
- **Effect:** TUI silently diverges; original events
  (QueueChanged/NowPlayingChanged) are lost; only the synthesized
  `Notification` reaches the client.
- **Fix:** on Lagged, fetch `core.snapshot()` and emit a `Resync` event
  (or synthetic NowPlayingChanged + QueueChanged pair).

### H13. Unbounded library-event payloads x N subscribers [ ]
- **File:** `src/ipc/protocol.rs` L110-147
- **Trigger:** library load with N TUI subscribers.
- **Effect:** memory pressure scales N x payload x ring depth.
- **Fix:** replace bulk events with `LibraryVersionChanged(u64)`;
  clients pull via existing RPCs.

### H14. Non-atomic config save [x]
- **File:** `src/config/mod.rs` L239-253
- **Trigger:** crash, OOM, power-loss mid-write.
- **Effect:** `config.toml` truncates or empties, wiping credentials.
- **Fix:** write to `path.with_extension("toml.tmp")` then
  `std::fs::rename` onto `path`. Mirror `QueueSnapshot::save`.

### H15. `update_server_config` clobbers `PasswordFile` [x]
- **File:** `src/daemon/core.rs` L1499-1524
- **Trigger:** user has `password_file = "..."` in `config.toml`, then
  edits server settings in the TUI.
- **Effect:** plaintext password written into `config.toml` even though
  the user set up file indirection.
- **Fix:** when `password_file` is set, write the password to that file
  atomically and leave inline `Password` cleared.

## Fix order (planned)

1. H14 (atomic config save). Trivial, prevents credential wipe.
2. H3 (restore-queue race). Small, removes a startup data-loss path.
3. H4 (Direct-mode idle race). Extends the just-shipped Buffered gate,
   removes the remaining "random skip" path.
4. H5 (early-skip after Buffered stop). Small, same `prebuffer_loading`
   gate or a `position > 0.5` guard.
5. H1 + H2 (state TOCTOU). Combine read+write into one critical
   section in both `play_queue_position` and the gapless-advance
   branch.
6. H7 (mpv command deadline). Bound `send_command` time; reader task
   refactor optional follow-up.
7. H9 (PipeWire cache divergence). Drop the cache short-circuit.
8. H11 (Replace + play atomicity). Daemon-side helper that takes a
   `Child` clone.
9. H15 (PasswordFile clobber). Handler routes password to the right
   destination.
10. H12 (broadcast resync), H13 (library version events), H10 (per-
    connection request ordering), H6, H8 deferred until after the
    above.
